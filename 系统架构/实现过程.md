## 2. 路由设计（Vue Router）

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | Home | 首页：推荐影片 + 热门排行 |
| `/movies` | movie/List | 影片列表，支持类型/地区/年份筛选 |
| `/movie/:id` | movie/Detail | 影片详情 + 影评 + 相似推荐 |
| `/cinemas` | cinema/List | 影院列表，按区域筛选 |
| `/order/seat/:showId` | order/Seat | SVG 选座 + 15分钟倒计时 |
| `/order/confirm/:orderId` | order/Confirm | 订单确认（积分/支付） |
| `/orders` | order/List | 我的订单列表 |
| `/login` | user/Login | 登录页 |
| `/register` | user/Register | 注册页 |
| `/user/profile` | user/Profile | 个人资料编辑 |
| `/user/member` | user/Member | 会员中心 |
| `/admin/dashboard` | admin/Dashboard | 数据看板 |
| `/admin/movies` | admin/MovieManage | 影片管理 |
| `/admin/orders` | admin/OrderManage | 订单管理 |
| `/admin/users` | admin/UserManage | 用户管理 |

### 路由守卫逻辑

```js
// router/index.js
router.beforeEach(async (to, from, next) => {
  const userStore = useUserStore()

  // 需要登录的路由：先拉最新用户信息，再判断
  if (to.meta.requiresAuth) {
    if (!userStore.token) {
      return next({ path: '/login', query: { redirect: to.fullPath } })
    }
    // 确保 userInfo 是最新的（防止刷新后误判）
    if (!userStore.userInfo) {
      await userStore.fetchUserInfo()
    }
    // admin 路由额外校验角色
    if (to.meta.requiresAdmin && userStore.userInfo.role !== 'admin') {
      return next('/')
    }
  }
  next()
})
```

## 3. 状态管理（Pinia）

### userStore — 全局持久
```js
{
  token: '',              // 从 httpOnly cookie 或 localStorage 读取
  userInfo: null,         // { id, phone, email, nickname, avatar, role, level, points }
  isLogin: computed,      // !!token
  login(credentials),     // 调用 POST /login，存 token
  logout(),               // 清 token，跳登录
  fetchUserInfo(),        // GET /profile
  updateProfile(data),    // PUT /profile（白名单字段）
  fetchMember(),          // GET /member
}
```

### movieStore — 影片缓存
```js
{
  hotList: [],
  recommendList: [],
  currentMovie: null,
  fetchHot(), fetchRecommend(), fetchDetail(id)
}
```

### 选座状态 — 不用 Pinia，用 sessionStorage

选座锁定状态本质是短暂 UI 状态，不适合放全局 Store。选座流程改用：

```
Seat.vue（本地 state）
  → 锁座成功后，将 { showId, seats, lockToken, expireAt } 写入 sessionStorage
  → 跳转 /order/confirm/:orderId
Confirm.vue
  → 从 sessionStorage 读取选座数据（防刷新丢失）
  → 也可通过 GET /order/:id 从后端恢复
```

## 4. API 接口

### request.js — Axios 实例

```js
// baseURL: '/api'
// withCredentials: true（配合 httpOnly cookie）
// 请求拦截器：从 cookie/localStorage 取 token，注入 Authorization 头
// 响应拦截器：401 → 清 token → 跳 /login；统一解包 response.data
```

### 接口清单

**用户模块** (`/api/user`)
- `POST /register` — 注册（手机号/邮箱 + 密码，密码长度≥8且含字母+数字）
- `POST /login` — 登录，返回 token
- `GET /profile` — 获取用户资料
- `PUT /profile` — 更新资料（白名单：nickname, avatar, email, phone。禁止修改 role/points）
- `PUT /password` — 修改密码
- `GET /member` — 会员信息（等级/积分/权益）

**影片模块** (`/api/movie`)
- `GET /list?type=&region=&year=&page=` — 影片列表
- `GET /:id` — 影片详情
- `GET /:id/reviews` — 该影片的影评列表
- `POST /:id/review` — 发布影评（需登录）

**影院模块** (`/api/cinema`)
- `GET /list?region=` — 影院列表
- `GET /:id` — 影院详情（含影厅列表）
- `GET /:id/schedule?movieId=&date=` — 该影院的排片

**订单模块** (`/api/order`)
- `POST /create` — 创建订单（body: { showId, seats, lockToken }）
- `POST /:id/pay` — 支付（body: { payToken, pointsUsed }，按钮点击后立即 disabled 防重复）
- `POST /:id/cancel` — 取消/释放座位
- `GET /:id` — 订单详情（后端校验 user_id 归属，含取票码）
- `GET /list?status=` — 我的订单列表
- `POST /lock-seats` — 锁座（body: { showId, seats }，返回 { lockToken, expireAt }）

**推荐模块** (`/api/recommend`)
- `GET /home` — 首页个性化推荐
- `GET /similar/:movieId` — 相似影片推荐
- `GET /hot` — 热门排行

**后台管理** (`/api/admin`) — 所有接口后端校验 role=admin
- `GET /dashboard` — 数据看板统计
- `GET /movies` — 影片列表（管理用）
- `POST /movies` — 新增影片
- `PUT /movies/:id` — 编辑影片
- `DELETE /movies/:id` — 删除影片
- `GET /schedules` — 排片列表
- `POST /schedules` — 新增排片
- `GET /orders?status=&page=` — 订单查询
- `POST /orders/:id/refund` — 退款审核
- `GET /users?status=&page=` — 用户管理
- `PUT /users/:id/status` — 封禁/解封

## 5. 订单状态机

```
PENDING ──支付──→ PAID ──退款──→ REFUNDED
   │                  
   ├──取消──→ CANCELLED
   │                  
   └──超时──→ EXPIRED（前端倒计时归零 → 调 cancel → 后端定时任务兜底清理过期锁）
```

## 7. 安全规约

| 问题 | 措施 |
|------|------|
| Token 存储 | 优先 httpOnly cookie；如用 localStorage，配 CSP 禁止内联脚本 |
| Token 校验 | Axios 拦截器对空 token 直接跳登录 |
| 越权访问 | 订单/用户资料等接口后端必须加 `WHERE user_id = :currentUserId` |
| 支付幂等 | 订单创建时生成 `payToken`（UUID），后端做 UNIQUE 约束，同一 payToken 只支付一次 |
| 支付防重复 | 前端 pay 按钮点击后立即 disabled，请求完成前不可再次点击 |
| 锁座竞态 | 锁座返回 `lockToken`，创建订单时校验 lockToken；定时任务每30秒清理过期锁 |
| 提权防护 | `PUT /profile` 白名单字段（nickname/avatar/email/phone），role 只能数据库手动改 |
| 管理员校验 | 后端 `/admin/*` 每个接口都查 role=admin，前端守卫仅 UX 优化 |
| 密码校验 | 前端+后端双重校验：长度≥8、含字母+数字 |
| XSS | 避免 `v-html` 用于用户生成内容（Vue `{{ }}` 默认安全） |

## 8. 骨架搭建步骤

1. `npm create vite@latest frontend -- --template vue` — 创建 Vue3 + Vite 项目
2. `npm install vue-router pinia element-plus axios` — 安装依赖
3. 创建目录结构和空文件（按上述结构）
4. 配置 `main.js`（注册 Router/Pinia/ElementPlus）
5. 配置 `router/index.js`（路由表 + 守卫，先 await fetchUserInfo 再判角色）
6. 配置 `api/request.js`（Axios 封装 + 拦截器 + 防重复提交）
7. 创建 `App.vue`（NavBar + router-view 布局）
8. 创建 `NavBar.vue` 公共组件
9. 创建 `stores/user.js`、`stores/movie.js`（Pinia 骨架）
10. 创建 `utils/index.js`（token/sessionStorage 工具函数）
11. 创建各 `views/*.vue` 占位页面（含 `<template>` 骨架）
12. 创建 `components/` 公共组件骨架
13. 验证 `npm run dev` 可运行，所有路由可跳转
